Higher-Order Functions
If functions can accept other functions as arguments and produce func-
tions as return values, that language is said to support
higher-order
functions
.
Higher-order functions are an essential part of functional programming languages.
In previous atoms, we’ve seen higher-order functions such as
filter()
,
map()
, and
any()
.
If you declare a function that takes a parameter of a function type, you can pass a
function (including a lambda) as an argument. You can also store a lambda. Let’s
look at the type of this storage:
