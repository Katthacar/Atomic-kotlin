Kotlin accepts this code, but what does it mean to the reader? Because code is read
much more than it is written, you should make your programs as understandable as
possible.
Languages work just fine without operator overloading. It isn’t an essential feature,
but is an excellent example of how a language is more than just a way to manipulate
the underlying computer. That’s the easy part. The hard part is crafting the language
to provide better ways to express your abstractions, so humans have an easier time
understanding the code without getting bogged down in needless detail. It’s possible
to define operators in ways that obscure meaning, so tread carefully.
