Invoke
Placing parentheses after an object generates a call to
invoke()
, so the
invoke()
operator makes an object look like it’s a function. You can define
invoke()
with any
number of parameters:
// OperatorOverloading/Invoke.kt

atomictest.eq
class
Func
{
operator fun
invoke
() =
"invoke()"

invoke
(i: Int) =
"invoke($i)"
operator fun
invoke
(i: Int, j: String) =
"invoke($i, $j)"
operator fun
invoke
(
i: Int, j: String, k: Double
)=
"invoke($i, $j, $k)"
}
fun
main
() {

f = Func()
f() eq
"invoke()"
f(
22
) eq
"invoke(22)"
f(
22
,
"Hi"
) eq
"invoke(22, Hi)"
f(
22
,
"Three"
,
3.1416
) eq
"invoke(22, Three, 3.1416)"
}
