3. The standard library included with a Kotlin application is small, because it

Making a collection read-only makes the collection safer and more bug-free by
preventing accidental modification.
Java provides a partial solution for collection immutability: when returning a
collection you can place it inside a special wrapper that throws an exception for
any attempt to modify the underlying collection. This doesn’t produce static type
checking, but can still prevent subtle bugs. However, you must remember to wrap
the collection, whereas in Kotlin you must be explicit when you
want
a mutable
collection.
