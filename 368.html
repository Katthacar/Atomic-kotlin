Summary
Languages like C++ and Java allow inheritance unless you specifically disallow it.
Kotlin assumes that you
won’t
be using inheritance—it actively prevents inheritance
and polymorphism unless they are intentionally allowed using the
open
keyword.
This provides insight into Kotlin’s orientation:
Often, functions are all you need. Sometimes objects are very useful.
Objects are one tool among many, but they’re not for everything.
If you’re pondering how to use inheritance in a particular situation, consider whether
you need inheritance at all, and apply the maxim
Prefer extension functions and
composition to inheritance
(modified from the book
Design Patterns⁹⁵
).
