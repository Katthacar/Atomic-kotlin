Recovery
The ambition of exception handling is recovery. This means that you fix the problem,
return the program to a stable state, and resume execution. Recovery often includes
Logging
information about the error.
Quite often, recovery isn’t possible. An exception might represent an unrecoverable
program failure, either a coding error or something uncontrollable in the environ-
ment.
