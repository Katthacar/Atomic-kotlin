When you call

Parameterized Types
We consider it good practice to use type inference—it tends to make the code cleaner
and easier to read. Sometimes, however, Kotlin complains that it can’t figure out what
type to use, and in other cases explicitness makes the code more understandable.
Here’s how we tell Kotlin the type contained by a
List
:

atomictest.eq
fun
main
() {
// Type is inferred:
val
numbers = listOf(
1
,
2
,
3
)
val
strings =
listOf(
"one"
,
"two"
,
"three"
)
// Exactly the same, but explicitly typed:
val

1
,
2
,
3
)
val

listOf(
"one"
,
"two"
,
"three"
)
numbers eq numbers2
strings eq strings2
}

numbers
contains a
List
of
Int

while

contains a
List

String
