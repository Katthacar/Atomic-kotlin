We’ve talked about identifiers like
house
and
sofa
as if they were objects. They are
actually
references
to objects. One way to see this is to observe that two identifiers
can refer to the same object:
// Properties/References.kt
class
Kitchen
{
var
table: String =
"Round table"
}
fun
main
() {
val
kitchen1 = Kitchen()
val
kitchen2 = kitchen1
println(
"kitchen1: ${kitchen1.table}"
)
println(
"kitchen2: ${kitchen2.table}"
)
kitchen1.table =
"Square table"
println(
"kitchen1: ${kitchen1.table}"
)
println(
"kitchen2: ${kitchen2.table}"
)
}
