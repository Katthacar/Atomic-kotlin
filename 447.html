The failure message is not as explicit as the two-parameter version, but in some cases
it is sufficient.
requireNotNull()
requireNotNull()
tests its first argument and returns that argument if it is non-
null
. Otherwise, it produces an
IllegalArgumentException
.
Upon success,
requireNotNull()
’s argument is automatically smart-cast to a non-
null
type. Thus, you usually don’t need
requireNotNull()
’s return value:
