Similar functions are typically included in every unit test framework, but the names
and parameter order can be different. For example, the
message
parameter in
assertEquals()
might be first or last. Also, it’s easy to mix up
expected
and
actual
—using named arguments avoids this problem.
The
expect()
function in
kotlin.test
runs a block of code and compares that
result with the expected value:
fun
<
T
> expect(
expected: T,
message: String?,
block: () -> T
){
assertEquals(expected, block(), message)
}
