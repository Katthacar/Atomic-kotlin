Kotlin imposes an additional constraint when overriding functions. Just as you
cannot inherit from a base class unless that base class is
open
, you cannot
override
a function from a base class unless that function is defined as
open
in the base
class. Inheritance and overriding cannot be accomplished in Kotlin without clear
intentions.
It’s especially interesting to take a
Bonobo

Chimpanzee

ordinary
GreatApe

talk()

Inside
talk()

GreatApe

talk()
’s

GreatApe

Often when you override a function, you want to call the base-class version of that
function (for one thing, to reuse the code), as seen in the overrides for
eat()

produces a conundrum: If you simply call
eat()

currently inside (as we’ve seen in
Recursion

eat()
