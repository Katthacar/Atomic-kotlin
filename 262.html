Kotlin unfolds the constructor reference
::Student
into the lambda:
{ index, name -> Student(index, name) }
mapIndexed()
was introduced in
Lambdas
. It turns each element in
names
into the
index of that element along with the element. In the definition of
students
, these
are explicitly mapped into the constructor, but the identical effect is achieved with
names.mapIndexed(::Student)
. Thus, function and constructor references can
eliminate specifying a long list of parameters that are simply passed into a lambda.
Function and constructor references are often more readable than lambdas.
