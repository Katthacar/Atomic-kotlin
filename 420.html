A
Counter
keeps track of a
count
and returns the next
Int
value.
new()
,
new2()
and
new3()
each create a different implementation of the
Counter
interface.
new()
returns an instance of a named inner class,
new2()
returns an instance of an
anonymous inner class, and
new3()
uses a
SAM conversion
to create an anonymous
object. All the resulting
Counter
objects have implicit access to the elements of the
outer object, thus they are inner classes and not just nested classes. You can see from
the output that
count
in
CounterFactory
is shared by all
Counter
objects.
SAM conversions are limited—for example, they do not support
init
clauses.
In Kotlin, files can contain multiple top-level classes and functions. Because of this,
there’s rarely a need for local classes, so if you do need them they should be basic
and straightforward. For example, it’s reasonable to create a simple
data
class that’s
only used inside a function. If a local class becomes complex, you should probably
take it out of the function and make it a regular class.
