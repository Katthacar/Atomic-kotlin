Companion Objects
Member functions act on particular instances of a class. Some functions
aren’t “about” an object, so they don’t need to be tied to that object.
Functions and fields inside
companion object
s are about the class. Regular class
elements can access the elements of the companion object, but the companion object
elements cannot access the regular class elements.
As you saw in
Objects
, it’s possible to define a regular
object
inside a class, but
that doesn’t provide an association between the
object
and the class. In particular,
you’re forced to explicitly name the nested
object
when you refer to its members.
If you define a companion object inside a class, its elements become transparently
available to that class:
