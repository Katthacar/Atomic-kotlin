Only members of the same class can access

The
private
keyword means no one can access that member except other members
of that class. Other classes cannot access
private
members, so it’s as if you’re also
insulating the class against yourself and your collaborators. With
private
, you can
freely change that member without worrying whether it affects another class in
the same package. As a library designer you’ll typically keep things as
private
as
possible, and expose only functions and classes to client programmers.
Any member function that is a
helper function
for a class can be made
private
to ensure you don’t accidentally use it elsewhere in the package and thus prohibit
yourself from changing or removing that function.
The same is true for a
private
property inside a class. Unless you must expose
the underlying implementation (which is less likely than you might think), make
properties
private
. However, just because a reference to an object is
private
inside

public
reference to the same
