This implementation is terribly inefficient because the previously-calculated results
are not reused. Thus, the number of operations grows exponentially:
Inefficient Computation of Fibonacci Numbers
When computing the 50th Fibonacci number, we first compute the 49th and 48th
numbers independently, which means we compute the 48th number twice. The 46th
number is computed as many as 4 times, and so on.
