The functions
f()
and
g()
and the property
n
have identical signatures in interfaces
A
and
B
, so Kotlin doesn’t know what to do and produces an error message if you
don’t resolve the issue (try individually commenting the definitions in
C
). Member
functions and properties can be overridden with new definitions as in
f()
, but
functions can also access the base versions of themselves using the
super
keyword,
specifying the base class in angle brackets, as in the definition of
C.g()
and
C.n
.
Collisions where the identifier is the same but the type is different are not allowed
in Kotlin, so you cannot resolve them.
